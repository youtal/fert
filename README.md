# Fret 框架 - 高性能粒子演化与数独解算系统 (v0.2.0)

Fret 是一个基于 **Vue 3** 构建的高性能 Web 应用程序，致力于在浏览器端实现复杂物理系统、生态博弈的可视化模拟以及高并发算法解算展示。

## 📖 项目简介

Fret 框架旨在探索 Web 渲染与前端计算的极限，当前集成了两大核心模块：

### 1. 粒子演化仿生系统
利用**空间哈希 (Spatial Hashing)** 物理优化算法，在维持数百高频交互粒子时，依然保持极低的 CPU 占用与流畅的帧率。
- **猎物 (Preys)**：遵循 Boids 群集算法，寻找同伴并躲避捕食者。
- **捕食者 (Predators)**：追踪最近的猎物，若长时间未进食则会因饥饿而“过载死亡”。
- **进化博弈**：种群在繁衍、捕食与突变间寻求动态平衡。

### 2. 启发式数独解算引擎
基于纯前端构建的高性能数独求解与生成演示系统。
- **状态向量 (Bitmasks)**：使用位运算（Uint16Array）记录行列块可用状态，极大提升查询效率。
- **约束传播 (Constraint Propagation)**：基于队列的工作流逻辑解算，识别唯一余数直接填入（即时标绿）。
- **启发式暴力回溯 (MRV Backtracking)**：在逻辑无法继续推进时，自动选取剩余候选数最少的单元格（黄色高亮）进行深度优先搜索，回溯时复用状态向量避免冗余计算。

## 🏗️ 技术架构

项目采用了清晰的层次化设计方案：

-   **渲染与交互层 (Views/Components)**: 提供玻璃拟态 (Glassmorphism) 风格的深色 UI，支持动态状态跟踪与解算全过程可视化动画。
-   **模型与算法层 (Models/Utils)**: 封装了核心算法（空间索引、转向力；位运算状态机、DFS 回溯）。
-   **逻辑层 (Composables)**: 负责驱动物理循环与渲染管道。
-   **状态层 (Stores)**: 基于 `Pinia` 实现全局状态持久化与日志。

## 📦 安装与运行

### 环境要求
- Node.js: `^20.19.0` 或 `>=22.12.0`
- NPM/PNPM

### 安装步骤
```bash
# 克隆并进入目录
cd fret

# 安装依赖
npm install
```

### 运行命令
```bash
# 启动开发服务器 (Vite)
npm run dev

# 执行单元测试
npm run test:unit

# 构建生产环境版本
npm run build

# 格式化代码
npm run format
```
